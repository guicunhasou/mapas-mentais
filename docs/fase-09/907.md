# 907
## Connection APIs

- Introdução às APIs
  - Definição: Regras e protocolos de comunicação
  - Função: Intermediária entre cliente e servidor
  - Formato de dados: Geralmente JSON
  - Exemplos: APIs de Navegador e de Terceiros

- Tipos de APIs
  - APIs Internas
    - Uso: Integração organizacional própria
    - Vantagens: Segurança e personalização total
    - Desvantagens: Manutenção e custo de infraestrutura

  - APIs Externas
    - Uso: Funcionalidades de terceiros (Mapas, Pagamentos)
    - Vantagens: Inovação e redução de esforço interno
    - Desvantagens: Dependência e custos de licença

- Gerenciamento e Boas Práticas
  - Controle de acesso e autenticação
  - Monitoramento de desempenho e erros
  - Documentação completa para desenvolvedores
  - Uso de variáveis de ambiente (.env)

- Ferramentas de Consumo (React Native)
  - Axios
    - Baseado em Promises
    - Tratamento automático de erros
    - Suporte a interceptores e instâncias globais

  - Fetch API
    - Nativa do JavaScript
    - Leve e sem instalação extra

- Métodos HTTP
  - GET: Recuperar dados (Idempotente)
  - POST: Enviar novos dados (Cria recurso)
  - PUT: Atualizar recurso existente (Substituição)
  - DELETE: Remover recurso do servidor

- Projeto Prático e Renderização
  - Hooks: useState e useEffect para dados
  - Componente FlatList: Listas eficientes
  - Estrutura: Pasta components, screens e services
  - Interface IStore: Tipagem forte com TypeScript
